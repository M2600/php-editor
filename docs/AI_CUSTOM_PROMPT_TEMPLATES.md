# AIツール使用ガイド - カスタムプロンプト用テンプレート

このファイルは、AIチャットの「カスタムプロンプト」機能で使用できるプロンプトテンプレート集です。
コピー&ペーストして、AI設定画面の「カスタムプロンプト」欄に貼り付けてください。

---

## テンプレート1: 効率重視（推奨）

```
# ツール使用の鉄則
1. まず `searchFiles` で検索して対象を絞り込む
2. 次に `readFile` で構造のみ取得（パラメータなし）
3. 必要な部分のみ `readFile` で行範囲指定して取得
4. 編集は最小限の変更で、変更後は必ず確認

# 禁止事項
- いきなり大きなファイルを全て読み込む
- 構造確認せずに編集する
- 同じファイルを何度も読み込む

# 必ず守ること
- 100行超のファイルは段階的に読み込む
- 検索を最優先で活用する
- コンテキストを節約する
```

---

## テンプレート2: 詳細説明重視

```
# ツール使用ワークフロー
## 調査時
1. searchFiles: ファイル名や関数名を検索
   - 例: searchFiles("class UserManager", searchIn="content")
2. readFile（構造のみ）: クラス・関数一覧を取得
   - 例: readFile("api/user.php")
3. readFile（行範囲指定）: 必要な部分のみ取得
   - 例: readFile("api/user.php", startLine=20, endLine=50)

## 編集時
1. readFileで現在の内容を確認
2. editFileByReplace または editFileByLines で編集
3. readFileで変更を確認

## 効率化
- ファイルが100行超 → 構造確認してから部分読み込み
- 複数ファイル → 1つずつ段階的に
- 不要な情報は読み込まない
```

## テンプレート4: バグ修正特化

```
# バグ修正時のツール使用
1. searchFiles でエラーメッセージや関数名を検索
   - regex=true で正規表現検索可能
   - 例: searchFiles("error|exception", regex=true, filePattern="*.log")
2. 該当ファイルの構造を readFile で確認
3. 問題箇所のみ行範囲指定で読み込み
4. 周辺コードも確認してから修正
5. 修正後は必ず動作確認用のコードも確認

# 注意点
- ログファイルが大きい場合は searchFiles で絞り込む
- エラーハンドリングコードも確認する
- 関連ファイルへの影響も考慮
```

## テンプレート5: 新機能実装特化

```
# 新機能実装時のツール使用
1. searchFiles で類似機能を検索
   - 例: searchFiles("class UserManager", searchIn="content")
2. 既存コードの構造を readFile で把握
3. 参考にする部分を行範囲指定で読み込み
4. createFile で新ファイル作成
5. ls でディレクトリ構造を確認

# ベストプラクティス
- 既存コードのパターンに従う
- 段階的に実装して動作確認
- 関連ファイルとの整合性を保つ
```

## テンプレート6: コードレビュー特化

```
# コードレビュー時のツール使用
1. readFile で全体構造を把握
2. 気になる部分を行範囲指定で詳細確認
3. searchFiles で類似パターンを検索
4. セキュリティやパフォーマンスの観点で分析

# レビュー観点
- セキュリティ（XSS, SQLインジェクション等）
- パフォーマンス（N+1問題、不要な処理等）
- 可読性（命名、コメント等）
- 保守性（重複コード、結合度等）
```

## 使い方

1. 上記のテンプレートから目的に合うものを選ぶ
2. 必要に応じてカスタマイズ
3. AI設定画面の「カスタムプロンプト」欄に貼り付け
4. 「カスタムプロンプトを使用」をONにする
5. 設定を保存

## 注意事項

- カスタムプロンプトはベースプロンプトに**追加**されます
- 矛盾する指示は避けてください
- 長すぎるプロンプトはコンテキストを圧迫します（500文字程度推奨）
- 定期的に見直して最適化してください

## 組み合わせ例

効率重視 + バグ修正特化:
```
# 効率的なバグ修正
1. searchFiles でエラー箇所を特定
2. readFile で構造のみ確認
3. 問題箇所を行範囲指定で読み込み
4. 最小限の変更で修正

禁止: 大きなファイルを全て読む、検索せずに総当たり
```
